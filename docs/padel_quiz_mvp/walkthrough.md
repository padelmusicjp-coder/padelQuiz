# PadelQuiz MVP - 修正内容の確認 (Walkthrough)

## 実装完了サマリー
PadelQuiz MVPとして、スマホブラウザに最適化された軽量なクイズアプリケーションの実装を完了しました。

### 主要な実装内容
- **React (Vite) + TypeScript** によるプロジェクト基盤の構築
- **CSS** によるレスポンシブでクリーンなUI実装（各種アニメーション、トランジション込）
- **状態管理 (useQuiz hook)** を用いた、タイトル → 問題 → 解説 → 結果 の画面遷移の実装
- **問題データ (questions.json)** の分離（テスト用にパデルのルール・用語・戦術に関する問題を5問用意）
- **AudioContext** を用いた、各画面でのタップ音・正解音・不正解音の実装（iOS等のモバイル環境制限に対応）

## 🎧 Phase 10-11: 動的BGM・音量オプションの追記
ゲームの没入感とユーザビリティを高めるため、オーディオシステムを大幅に強化しました。

### 🔈 トップ画面＆音量設定
- **トップ画面の静音化**: アクセス直後に音が鳴るのを防ぐため、トップ画面でのBGM再生を廃止し、「※ゲームを開始すると音が鳴ります。音量にご注意ください」という注意喚起テキストを追加しました。
- **⚙️ オプション機能**: タイトル画面からポップアップ（モーダル）を開き、**BGM音量**と**SE（効果音）音量**をそれぞれ「大・中・小・なし」の4段階で個別に調整できるようにしました。
- 設定した音量データはブラウザの `localStorage` に自動保存され、次回アクセス時も引き継がれます。電車内でのプレイなども安心です。

### 🎵 状況に応じた動的BGM（クロスフェード）
- **Chance & Pinch システム**: ゲームの正答状況や「連続ミス」をフックにして、プレイヤーがピンチ（焦っている）状態かどうかを `isPinch` フラグで判定します。
- 状態が変化したタイミングで、BGMが即座に切り替わるのではなく、現在流れている曲が滑らかにフェードアウトし、新しい曲がフェードインする「クロスフェード処理」を自前で実装しました。
- 結果画面（Result）に突入すると同時にゲーム内BGMはスムーズに無音へフェードアウトし、最終スコアに応じた専用ジングル（Perfect/Medium/Lowの3種類）が鳴り響く仕組みになっています。
