# PadelQuiz MVP 実装計画

## 目標
PadelQuizのMVP（最小実装）として、スマホブラウザに最適化され、軽量でテンポの良いクイズ体験を提供するWebアプリケーションを開発する。正解/不正解時の解説を中心とし「初心者を否定しない」設計とする。

## User Review Required
- **使用言語について**: より堅牢な開発ができるため、TypeScript + React (Vite) での構築を推奨しますがよろしいでしょうか？（不要な場合は通常のJavaScriptにも変更可能です）
- **UIテイストについて**: 本編の `padel-minimatch` (サイバー・ネオン風) のデザインを引き継ぎますか？ それとも、パデル初心者が親しみやすい、明るくポップ（あるいはクリーン）な全く別の世界観のデザインにしますか？
- **問題データについて**: 動作確認用として、テスト用ダミーのパデルクイズを数問JSONで作成して組み込みますが、よろしいでしょうか？

## Proposed Changes

### フロントエンド基盤
- **技術選定**: React (Vite) を採用。状態管理とコンポーネント分割を実現。
- **言語**: TypeScript または JavaScript

#### [NEW] `package.json` 等
Vite での初期化によるファイル群。

### データ構造

### コンポーネント構成
ディレクトリ: `src/components/`
- `TitleScreen`: タイトルと「スタート」ボタン。このボタン押下時に AudioContext を初期化する。
- `QuizScreen`: 進行状況（〇問目/全〇問）、問題文、選択肢。
- `ExplanationScreen`: 正解/不正解の表示と簡潔な解説。
- `ResultScreen`: 正解数に応じたランク表示（例：パデル初心者、レベル1等）と「再挑戦」ボタン。
- `QuizContainer`: 上記画面のルーティング・状態遷移を管理する親コンポーネント。

### 保存機能 (localStorage)
プレイ結果を `localStorage` に保存する（任意要件含む）。

## 追記機能：CSVによる問題データ管理

ユーザーがアプリの中ではなく、Excel等の表計算ソフト（CSVファイル）で問題データを管理し、それをJSONに一括変換できる仕組み（インポート／エクスポート）を構築します。

### User Review Required
以下のCSVのカラム（列）構成で問題ないかご確認ください。最大4つの選択肢に対応させ、正解には「TRUE」、不正解には「FALSE」を入力する形式です。

| id | categoryId | difficulty | question | explanation | tags | c1_text | c1_isCorrect | c2_text | c2_isCorrect | c3_text | c3_isCorrect | c4_text | c4_isCorrect |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| q_001 | rules | 1 | パデルのサーブは… | 腰より下で… | rules,serve | 選択肢A | TRUE | 選択肢B | FALSE | 選択肢C | FALSE | 選択肢D | FALSE |

### 開発内容
以下2つのNode.jsスクリプトを作成します。（アプリ自体には影響しません）
- **`scripts/export_csv.js`**
  現在の `src/data/questions.json` を読み込み、`questions.csv` を出力します。
- **`scripts/import_csv.js`**
  編集した `questions.csv` を読み込み、`src/data/questions.json` に上書きします。

CSVの確実な変換のため、開発環境のみで使用する軽量ライブラリ（`csv-parse`, `csv-stringify`）を使用します。

## 追記機能：ショット（スプライト）ごとのボール軌道の個別化

現在の「飛んでくる→打ち返す・空振り」という一連のボール軌道を、選ばれたキャラクターのショット種類（6種）に応じて細かく変動（CSSキーフレームで個別に定義）させます。

### 軌道要件案 (User Review Required)

以下の軌道定義で実装してもよろしいでしょうか？（※打球点となる「溜め」の0.7秒間とその後の動きを定義します）

| ショット (画像) | 飛んでくるボール（溜めの0.7秒） | 正解時（打ち返し） | 不正解時（空振り後） |
|---|---|---|---|
| **bandeja** (バンデージャ) | 高い位置から山なりでノーバウンドで飛んでくる | ノーバウンドで右斜め下へ山なり（ロブ風）に打ち返す | 高い位置からそのまま地面へ落ちて転がる |
| **cut** (カット) | 右斜め上からノーバウンドで直線的に飛んでくる | 低く鋭い直線軌道で右へ速く返す | そのままスルーして地面へ落ちる |
| **drop** (ドロップ) | 速い直線のボールがノーバウンドで向かってくる | ポンッと軽く当て、すぐ目の前に落ちて跳ねる（ドロップ） | そのままスルーして地面へ落ちる |
| **rebote** (レボテ) | 手前でバウンドし、キャラを通り過ぎて後ろ（左）の壁で跳ね返ってからキャラの位置に戻ってくる | 通常の山なり/直線で右へ打ち出す | 跳ね返ってきた球を見送り、足元に落ちる |
| **stroke** (ストローク) | 手前で一度バウンド（ワンバウンド）して向かってくる | 通常の鋭い直線で右へ返す | バウンドした球を空振りし転がる |
| **volley** (ボレー) | ノーバウンドで胸の高さに飛んでくる | ボレーとして直線的・あるいはやや下向きに返す | スルーして転がる |

### 開発内容
- `src/components/CharacterAnim.tsx` 内で、現在選択されているスプライト名（例：`stroke`, `volley` 等）を抽出し、ボールのCSSクラスを `<div className={\`padel-ball ball-${animState}-${shotType}\`} />` のように動的付与します。
- `src/index.css` に各ショット専用の `@keyframes` を定義します（`ball-incoming-bandeja`, `ball-hit-drop`, `ball-miss-rebote` 等、大量のキーフレームになりますがCSSで完結させます）。

## 追記機能：シチュエーションに応じた動的BGMシステム（Phase 10）

追加いただいた `bgm_chance` (5曲)、`bgm_pinch` (5曲)、および `bgm_title` (1曲) を活用し、プレイヤーの「状況（有利・不利）」に応じて音楽が切り替わるBGMシステムを実装します。

### User Review Required
ファイルの名称から、以下のような条件分岐でBGMを切り替える・あるいは選曲するロジックを提案します。いかがでしょうか？

#### 1. BGMの振り分けルール案
- **タイトル画面 および 結果画面**: `bgm_title.mp3` を再生。
- **ゲーム中（Quiz・Explanation画面）**:
  - **【Chance（有利）状態】**: クイズ開始時は、必ず `bgm_chance1〜5` の中から**ランダムで1曲**を選んで再生開始。
  - **【Pinch（不利）状態へ変化】**: クイズを進めている中で、「正答率が50%を下回る」または「2問連続で間違える」などの条件を満たしてピンチになると、現在の曲をフェードアウトし、`bgm_pinch1〜5` のいずれか（ランダム）へBGMが滑らかに移行する。
  - **【Chance（有利）状態へ復帰】**: ピンチ状態から正解を重ねて正答率を取り戻すと、再び `bgm_chance` 曲へ戻る。

#### 2. メモリ最適化（技術的な方針）
SE（効果音）は超低遅延で鳴らすためにメモリ上にデコードして展開していますが、BGM（数MB〜の尺が長いmp3が11曲）をすべてメモリ展開するとスマホ等のブラウザがクラッシュする恐れがあります。
そのため、BGMの再生には専用の `HTML5 <audio>` タグ制御（ストリーミング再生のような形）を使用し、クロスフェードなどもこの仕組みの上で実装します。

### 開発内容
- `src/utils/audio.ts` の拡張: BGM専用の Audio 要素プールを作成し、`playBGM(type)`・`fadeOut()`・`crossFade(newType)` メソッドを実装。
- `useQuiz.ts` への状態追加: 過去の解答履歴から `isPinch` フラグを計算するロジックを追加。
- BGMの切り替え: スコア状況が変化し `isPinch` が切り替わったタイミングで曲をクロスフェードさせる。
